# XRPL OnTheTrack Terminal — Judge Q&A

## 1. What problem are you solving?

XRPL has powerful tools, but new users often meet them as separate doors: wallets, onramps, DEX routes, bridges, Xahau, NFTs, issued currencies and DeFi concepts.

The problem is not lack of tools. The problem is missing route clarity, education and risk awareness before action.

## 2. What is your solution?

XRPL OnTheTrack Terminal is a guided XRPL education and proof layer.

It helps users:

1. learn the route
2. understand risk
3. connect wallet only when needed
4. create Xaman payloads
5. verify XRPL transaction hashes
6. record progress through XP or optional Proof Stamps

## 3. Is this a wallet?

No.

The Terminal is not a wallet. It works as an education, routing and proof layer around XRPL tools.

Wallet signing is handled through Xaman.

## 4. Do you custody user funds?

No.

The Terminal does not custody funds. It does not hold user assets or private keys.

## 5. Are you a broker or exchange?

No.

The Terminal does not act as broker, does not execute trades and does not exchange assets.

It routes users to official providers after education and risk awareness.

## 6. Does the app provide yield?

No.

The Terminal does not provide yield and does not promise returns.

If a partner route involves yield or DeFi concepts, the Terminal treats it as high-risk education and routes users to official providers only after explanation.

## 7. Why use SourceTag?

The SourceTag gives the project a clear XRPL proof identity.

```txt
2606170002
```

It lets the Terminal verify that a transaction belongs to the correct route or proof flow.

## 8. Why not put every action on-chain?

Because reading, clicking and learning do not need blockchain transactions.

The Terminal uses:

- off-chain XP for learning and engagement
- optional on-ledger Proof Stamps for meaningful completed routes

This keeps the user experience cleaner and avoids unnecessary ledger spam.

## 9. What is XP?

XP is an off-chain learning and engagement score.

XP has no guaranteed financial value. It is not a token. It is not an investment product.

Future token eligibility can only be reviewed after legal review.

## 10. Is there an OTT token?

The MVP includes testnet-only token simulation and eligibility logic.

Mainnet OTT token distribution stays locked until legal review is complete.

## 11. What is a Proof Stamp?

A Proof Stamp is an optional XRPL transaction that records that a user completed a meaningful route.

It can include:

- SourceTag `2606170002`
- memo data
- route name
- tx hash verification

## 12. What is Partner Hub?

Partner Hub is an education-first discovery layer for XRPL-related routes.

It can show:

- partner description
- risk level
- official links
- user checklist
- OTT angle
- optional proof stamp

## 13. What is Truth Desk?

Truth Desk is a paid education/service route.

It includes:

- Ask Truth: one short question
- 1-on-1 request: education or project feedback

Truth Desk does not provide financial, legal or tax advice.

## 14. What is Access Gate?

Access Gate is a terminal access route layer.

It supports concepts for:

- fiat access route through external provider
- XRP access payment
- RLUSD access route
- OTT Access Pass NFT

Access is utility access only, not investment.

## 15. Why use Xaman?

Xaman lets users sign XRPL payloads without the Terminal touching private keys.

The Terminal creates payloads and verifies the result.

## 16. What did you build during the hackathon?

The MVP includes:

- React/Vite/TypeScript terminal UI
- single Vercel API router
- Xaman payload creation
- XRPL transaction verification
- SourceTag proof checks
- Partner Hub
- Proof Stamp flow
- Truth Desk flow
- Access Gate flow
- Reward Ledger
- Pitch Mode
- Submission Pack

## 17. What is the business model?

Potential business routes:

- paid access route
- Ask Truth questions
- 1-on-1 education/project feedback
- partner discovery routes
- future access pass NFT
- future legal-reviewed token eligibility

All routes are positioned as utility, access or education — not investment.

## 18. What makes this different?

Most apps push users directly toward action.

XRPL OnTheTrack Terminal puts education and risk awareness before action.

It is not another wallet dashboard. It is a guided XRPL learning, routing and proof layer.

## 19. What are the main risks?

Main risks:

- regulatory interpretation
- user misunderstanding of token rewards
- partner route risk
- DeFi/yield misunderstanding
- payment route compliance

The MVP reduces this by using clear language:

- no custody
- no broker
- no yield provider
- no trade execution
- official provider routes
- mainnet token lock until legal review
