# XRPL OnTheTrack Terminal — Post-Hackathon Roadmap

## Purpose

This roadmap parks future ideas so they do not distract from the current MVP.

Current rule:

```txt
Finish MVP test and submission first.
Do not add these features before UI polish/demo/submission.
```

## Current MVP Focus

```txt
Vercel green
Live app review
Smoke Test pass
UI polish
Demo recording
Hackathon submission
```

## Roadmap Phase 1 — Stability After Submission

Goal:

```txt
Make the current MVP more stable and easier to use.
```

Tasks:


Every future feature must pass this question:


If yes, park it here.

If no, do not build it.
